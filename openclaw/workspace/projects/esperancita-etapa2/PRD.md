# PRD - Etapa 2: comportamento, atividades e criterios da Esperancita

## Objetivo

Subir e organizar todos os arquivos que definem comportamento, atividades, criterios operacionais, rotinas, limites, formatos de saida e responsabilidades da Esperancita.

## Pre-condicoes

- Etapa 1 concluida: identidade, memoria, mapa, GitHub e skills base.
- Workspace local validado.
- GitHub sincronizado.
- Segredos fora do versionamento.

## Entregaveis

- `SOUL.md` refinado com comportamento final.
- `AGENTS.md` refinado com regras de operacao.
- `USER.md` preenchido com contexto validado pelo usuario.
- `MEMORY.md` e topic files atualizados.
- Skills especificas em `skills/esperancita/`.
- Criterios de sucesso por atividade.
- Plano de crons e automacoes.
- Plano de deploy VPS, se SSH estiver pronto.

## Dados que o usuario precisa fornecer

- Contexto pessoal/profissional.
- Rotina e preferencias.
- Projetos ativos.
- Pessoas e contatos importantes.
- Atividades que a Esperancita deve executar.
- Atividades que a Esperancita nunca deve executar sem pedir.
- Nivel de proatividade aceitavel.
- Dominio/subdominio, se houver.

## Criterio de pronto

- Todos os arquivos comportamentais revisados.
- GitHub sincronizado.
- Teste via Telegram executado.
- Pendencias externas documentadas.
