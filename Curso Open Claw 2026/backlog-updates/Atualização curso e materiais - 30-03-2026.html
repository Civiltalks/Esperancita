🚨 Breaking Changes (requer ação nos materiais)

BREAKING PRD Remoção de nodes.run shell wrapper Unreleased
O wrapper de shell nodes.run foi removido do CLI e da tool nodes. Execução de shell em nodes agora vai exclusivamente via exec host=node. Capacidades específicas de node continuam em nodes invoke e ações dedicadas.
M8 Multi-Agentes M5 Integrações
📄 Ação recomendada — PRD
Atualizar materiais de M8 (multi-agentes) que mencionam nodes.run para shell remoto. Substituir exemplos por exec host=node. Verificar prompts que instroem uso de nodes tool para execução de comandos.
BREAKING PRD Task ledger JSON → SQLite (background tasks) Unreleased
O ledger JSON de tasks (tasks/runs.json) foi substituído por store SQLite. Qualquer script ou material que instrua alunos a ler esse arquivo diretamente vai quebrar. A interface suportada agora é via openclaw tasks.
M8 Multi-Agentes M7 Proatividade
📄 Ação recomendada — PRD
Verificar se algum material ou prompt do curso referencia tasks/runs.json diretamente. Se sim, substituir por openclaw tasks. Baixo risco — improvável que esteja nos materiais atuais, mas confirmar.
BREAKING PRD Remoção de qwen-portal-auth OAuth 2026.3.28
A integração OAuth qwen-portal-auth para portal.qwen.ai foi removida. Migração necessária para Model Studio com openclaw onboard --auth-choice modelstudio-api-key.
M1 Setup M5 Integrações
📄 Ação recomendada — NOTA DE GRAVAÇÃO
Impacto baixo para o curso — Qwen não é modelo recomendado no material atual. Verificar se alguma menção existe; se não, sem ação. Anotar como contexto caso aluno pergunte.
BREAKING PRD Config/Doctor: remoção de migrações antigas (>2 meses) 2026.3.28
Migrações automáticas de config mais antigas que 2 meses foram removidas. Chaves de configuração legadas muito antigas agora falham na validação em vez de serem reescritas automaticamente.
M1 Setup M2 Segurança
📄 Ação recomendada — NOTA DE GRAVAÇÃO
Alunos que instalaram na primeira versão do curso (Jan/2026) e nunca rodaram openclaw doctor podem ter configs inválidas após essa mudança. Recomendar que todos rodem openclaw doctor + openclaw config validate antes de prosseguir. Adicionar ao checklist do M1 ou M2.

🟡 Mudanças Importantes (analisar para materiais)

PROMPT DO ALUNO Before_tool_call hooks com requireApproval — aprovações via /approve 2026.3.28
Plugins podem agora pausar execução de tools e pedir aprovação do usuário. O comando /approve agora trata tanto exec approvals quanto plugin approvals, com fallback automático. Isso afeta o fluxo de aprovação que ensinamos no M2 (Immune System) e M9.
M2 Segurança M9 Immune System
✍️ Ação recomendada — PROMPT DO ALUNO
O sistema de /approve ficou mais unificado. Atualizar prompts/exemplos nos módulos de segurança para refletir que /approve agora cobre múltiplos tipos de aprovação. Oportunidade de simplificar o ensinamento: "um único comando para aprovar qualquer ação pendente".
PROMPT DO ALUNO Flows/tasks — parent flows para trabalho orquestrado Unreleased
Novo conceito de flows como entidade pai para ACP e subagents — trabalho orquestrado começa a ter um registro central. ACP e subagent updates agora roteiam pelo contexto de flow pai, emergindo no thread/session correto.
M8 Multi-Agentes
✍️ Ação recomendada — NOTA DE GRAVAÇÃO
Funcionalidade em desenvolvimento (Unreleased), mas sinaliza nova abstração importante. Monitorar para o módulo de multi-agentes — quando lançado, pode simplificar muito o ensino de orquestração. Por ora, apenas registrar como contexto futuro.
NOTA DE GRAVAÇÃO ACP current-conversation binds — Discord, BlueBubbles, iMessage 2026.3.28
/acp spawn codex --bind here agora funciona no Discord sem criar child thread — transforma o chat atual num workspace ACP. Muda o fluxo de uso do Codex via Discord que pode estar nos exemplos do curso.
M8 Multi-Agentes M5 Integrações
✍️ Ação recomendada — NOTA DE GRAVAÇÃO
Adicionar nota nos materiais de ACP/multi-agentes sobre o novo --bind here. Flexibilidade adicional para alunos que usam Discord como interface.
PRD Exec/runtime: host=auto por default — sandbox só quando existe Unreleased
Exec agora usa host=auto por default, resolvendo para sandbox apenas quando existe. Comportamento anterior podia confundir alunos em setup sem sandbox — agora falha fechado corretamente.
M1 Setup M2 Segurança
📄 Ação recomendada — PRD
Verificar se os materiais de setup mencionam comportamento de exec/sandbox. Mudança torna comportamento mais intuitivo — pode simplificar explicação de troubleshooting do M1.
PROMPT DO ALUNO WhatsApp Reactions — agentes podem reagir com emoji Unreleased
Agentes agora podem reagir com emoji em mensagens WhatsApp recebidas — feedback não-verbal mais natural (ex: ❤️ em vez de resposta de texto).
M5 Integrações M3 Identidade
✍️ Ação recomendada — PROMPT DO ALUNO
Boa feature para adicionar em exemplos de prompts de identidade e integrações WhatsApp. Torna agentes mais "humanos". Sugerir como melhoria nos prompts de SOUL.md dos alunos.
NOTA DE GRAVAÇÃO Cron/announce: payloads multi-linha entregues completos no Telegram Unreleased
Fix importante: relatórios de cron em modo announce agora entregam todos os chunks de texto no Telegram forum topics (tópicos). Bug anterior fazia truncar relatórios longos.
M7 Proatividade
