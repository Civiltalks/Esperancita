📅 Google Calendar (via gog CLI)

⭐
Novidade 3.2 — gog CLI recomendado

A partir da 3.2, o gog CLI é o caminho recomendado para Google Workspace. Um único comando configura Calendar, Drive, Gmail e Docs — sem criar projeto no Google Cloud Console manualmente.

# Instalar gog CLI (uma vez) npm install -g gog # Autenticar — um comando para tudo: # Calendar + Drive + Gmail + Docs gog auth login # Usar Calendar gog calendar list --account=SEU_EMAIL gog calendar create "Reunião produto" --time="amanhã 14h" # Usar Drive gog drive ls gog drive upload ./relatorio.pdf # Usar Gmail gog gmail inbox --limit 10 gog gmail send --to "fulano@email.com" --subject "..."
✅ Por que gog em vez do OAuth manual?
  • Sem criar projeto no Google Cloud Console
  • Sem configurar OAuth credentials manualmente
  • Um único gog auth login autentica todos os serviços Google
  • Skills de Calendar, Drive e Gmail já usam gog por padrão na 3.2

📄 PDF Tool Nativo NOVO 3.2

✨ Sem instalar nada — já funciona!

A partir da versão 3.2, agentes podem analisar documentos PDF nativamente. Basta mencionar o arquivo no chat.

# O agente simplesmente recebe o PDF e analisa: "Analise este contrato: /caminho/para/documento.pdf" "Resuma este relatório e extraia os KPIs principais" "Compare estes dois PDFs e me diga as diferenças"
  • Suporte nativo: Claude (Anthropic) e Gemini (Google)
  • Outros modelos: fallback automático via extração de texto
  • Limite: até 10 PDFs por mensagem

💡 Casos de uso reais: analisar boletos, extrair dados de NFs para o Notion, resumir relatórios longos, revisar contratos.
