Antes de Começar
Pré-requisitos e primeiros passos
📋 Pré-requisitos
  • Conta na Hostinger — hostinger.com
  • Conta na Anthropic com billing ativo — console.anthropic.com
  • Telegram instalado no celular
  • Cartão de crédito para pagamento da VPS
1
Criar a VPS na Hostinger
~3 min

Acesse hostinger.com/vps-hosting e escolha o plano KVM 1 — 1 vCPU, 4GB RAM, 50GB SSD. Mais que suficiente.

🚫
NÃO use o template Docker/One-Click do OpenClaw

Docker isola o agente num container — instalar skills e integrações fica muito mais complicado. Instale bare metal.

💡
Docker: Flag --container

Desde v2026.3.24, você pode usar openclaw --container <nome> para rodar comandos dentro de um container Docker/Podman existente. Útil para quem já tem infraestrutura containerizada.

Selecione Ubuntu 24.04, defina senha root forte, anote o IP da VPS.

2
Conectar via SSH
~2 min
ssh root@SEU_IP_DA_VPS

No Windows: use PuTTY ou Windows Terminal. Prefere terminal no navegador? hPanel → VPS → Terminal — funciona sem instalar nada.

3
Instalar o OpenClaw
~3 min
# Instalar OpenClaw (1 comando) curl -fsSL https://openclaw.ai/install.sh | bash

Depois, rodar o wizard de configuração:

openclaw onboard --install-daemon

O wizard vai perguntar: Gateway mode → Local | AI Provider → Anthropic | API Key → cole sua chave | Model → Claude Sonnet | Instalar como serviço? → Sim

Nota: O modelo padrão OpenAI agora é gpt-5.4 (desde v2026.3.22).

3.5
Configurar Perfil
de Ferramentas
Novo obrigatório na v2026.3.2 — sem isso o agente não faz nada útil

🚨 CRÍTICO — Não Pule Este Passo

A partir da versão 2026.3.2, o OpenClaw vem com tools.profile = messaging por padrão. O agente só consegue responder mensagens — NÃO pode executar comandos, ler arquivos, usar ferramentas ou fazer automações.

Para ter um agente verdadeiramente funcional, você PRECISA mudar para full.

# Configurar o perfil completo de ferramentas openclaw config set tools.profile full # Validar que a configuração está correta (novo na v2026.3.2) openclaw config validate

A saída do validate deve mostrar:

✅ tools.profile: full ✅ gateway.mode: local ✅ ai.provider: anthropic ✅ Configuration valid — 0 warnings
💡
Por que esse default mudou?

Muitas instâncias OpenClaw expostas na internet davam acesso total de ferramentas a qualquer pessoa que encontrasse o bot. O novo default messaging é mais seguro. Para o curso, queremos full — por isso este passo.

4
Verificar se está rodando
~30 seg
openclaw gateway status # Deve mostrar: running ✅
📡
Telegram Streaming — Novo na v2026.3.2

O streaming do Telegram agora é ativado por padrão. Você vai ver "digitando..." aparecer no Telegram enquanto o agente processa. É normal e significa que está funcionando!

5
Criar o Bot no Telegram
~3 min

Abra o Telegram → busque @BotFather → envie /newbot → escolha nome e username (deve terminar em "bot") → copie o token.

6
Conectar Telegram ao OpenClaw
~2 min
openclaw provider add telegram # Cole o token quando pedir

Depois abra o chat com seu bot no Telegram e envie /start.

ALERTA: dmPolicy aberto = risco grave

Se dmPolicy estiver "open", QUALQUER PESSOA que encontrar seu bot pode comandar seu agente. Verifique agora.

# Ver config atual cat ~/.openclaw/openclaw.json # Garantir dmPolicy = allowlist com seu Telegram ID # Descobrir seu ID openclaw gateway logs # aparece nas mensagens recebidas

Troubleshooting
Erros comuns e como resolver
❌ "Command not found: openclaw"
source ~/.bashrc # Ou reinstalar curl -fsSL https://openclaw.ai/install.sh | bash
❌ "API key inválida"
Verificar se tem billing ativo na Anthropic (console.anthropic.com → Billing). Copiar a key novamente sem espaços extras.
❌ "Agente responde mas não consegue executar comandos" (NOVO v2026.3.2)
O tools.profile ainda está como 'messaging'. Execute:
openclaw config set tools.profile full openclaw gateway restart
❌ "Bot não responde no Telegram"
openclaw gateway logs openclaw gateway status
❌ "Gateway não inicia"
ss -tlnp | grep 18789 openclaw gateway restart
