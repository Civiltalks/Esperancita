Como ligar o cérebro do seu agente

Uma aula modular para ativar memória operacional em três níveis: primeiro o cérebro pessoal, depois a diretoria, depois a equipe. Cada pessoa faz o bloco que corresponde ao seu papel.

Duração sugerida75–90 min, com blocos puláveis

Demo ao vivoFoco no pessoal; outros como upgrade

Tese da aula

Não é uma aula sobre Git. É uma aula sobre memória.

A arquitetura completa tem três cérebros, mas a adoção começa pelo ponto de menor atrito: a pessoa sentir que o agente dela ficou mais inteligente.

“Hoje você vai ligar o primeiro cérebro do seu agente. Depois, se fizer sentido para o seu papel, você evolui para diretoria e equipe.”

Bloco obrigatório

Cérebro pessoal

Para todos. Fundador, diretor, gestor, funcionário ou freelancer.

Promessa emocional: “meu agente lembra de mim.”

  • Contexto individual
  • Preferências e rotinas
  • Projetos pessoais
  • Primeira vitória rápida
Bloco liderança

Cérebro da diretoria

Só para founder, sócio, diretoria e liderança. Funcionário pode pular.

Promessa estratégica: “decisão sensível não fica perdida no WhatsApp.”

  • Dinheiro com nome
  • Pessoas e conflitos
  • Contratos e jurídico
  • Planejamento de alto nível
Bloco operação

Cérebro da equipe

Para founder, gestor, ops e colaboradores. Alguns criam; outros só contribuem.

Promessa operacional: “o time inteiro trabalha com a mesma memória.”

  • Inbox por pessoa
  • Áreas da empresa
  • Consolidação
  • Agentes com contexto real
Roteamento por perfil

Cada aluno segue um caminho diferente

Isso evita que um funcionário precise assistir diretoria e evita que um founder saia sem entender a arquitetura completa.

Funcionário
Faz: cérebro pessoal. Pula: diretoria. Assiste: equipe para entender como contribuir no inbox da empresa.
Gestor
Faz: pessoal + equipe. Diretoria: só assiste se tiver acesso a decisões sensíveis.
Founder
Faz tudo: pessoal → diretoria → equipe. É quem entende governança, privacidade e rollout.
Sócio/diretor
Faz: pessoal + diretoria. Equipe: faz se também for responsável por operação.
Roteiro da aula

75–90 minutos, com blocos independentes

O ideal é gravar como uma aula única, mas com marcações claras para a pessoa pular blocos sem se perder.

Mapa mental dos três cérebros

Explicar por que existe pessoal, diretoria e equipe. Mostrar que não é complexidade gratuita: é controle de contexto e permissão.

“O erro é tentar fazer um cérebro único para tudo. A empresa tem camadas de memória, assim como tem camadas de acesso.”
0–12 min

Bloco 1 — ligar o cérebro pessoal

Demo principal. É o único bloco que precisa ser realmente mão na massa para todo mundo.

  1. Criar repo a partir do template pessoal.
  2. Rodar inicialização.
  3. Abrir no Claude/Cowork/OpenClaw.
  4. Rodar /cerebro.
  5. Salvar a primeira memória.
  6. Testar recall: “o que você sabe sobre mim agora?”
“O primeiro uau não é a empresa organizada. É o seu agente responder como alguém que já te conhece.”
12–40 min

Bloco 2 — cérebro da diretoria

Bloco condicional. Ensinar conceito, regra dos 3 gatilhos e fluxo ideal. Só fazer demo se o repo estiver corrigido.

  1. Quando criar: mais de uma pessoa tomando decisão sensível.
  2. O que entra: dinheiro nominal, pessoas, contrato/jurídico.
  3. O que não entra: operação normal da equipe.
  4. Como distribuir: humano decide, não automação cega.
“Diretoria é default privado. Se você tiver dúvida se algo é sensível, ele começa aqui — não no cérebro do time.”
40–58 min

Bloco 3 — cérebro da equipe

Mostrar como a memória deixa de ser individual e vira operação. Aqui entram inbox, áreas, staging e consolidação.

  1. Cada pessoa captura no próprio fluxo.
  2. O que é público vai para o inbox da equipe.
  3. O agente consolida em áreas canônicas.
  4. O time passa a operar com contexto compartilhado.
“O cérebro da equipe não nasce perfeito. Ele nasce com inbox. A maturidade vem da consolidação.”
58–76 min

Fechamento — como isso vira rotina

Conectar com OpenClaw, agentes 24/7, revisão semanal e governança. Não prometer autonomia total no dia 1.

“Hoje você ligou a memória. A partir daqui, a pergunta deixa de ser ‘qual prompt eu uso?’ e vira ‘qual contexto o agente precisa para operar?’”
76–90 min

Prontidão dos templates

O que precisa ser corrigido antes de gravar

Essa tabela é para produção interna. Pode virar checklist do Vitor/time antes da aula.

Template pessoal Init roda, mas sobram placeholders como {{AGENTE_PESSOAL_NOME}}, {{ORG}}, {{SUA_CONTA}}. README promete skill inexistente setup-estacao-pessoal. Narrativa ainda está mais “sócio-diretor” do que “pessoal para todos”. Parcial
Template diretoria Init roda, mas há P0: setup-openclaw-pessoal-diretor não existe e a privacidade está contraditória. O repo diz default privado, mas templates de inbox falam público por default. Bloqueia demo
Template equipe Init roda, mas sobram placeholders operacionais e docs pós-init apontam para pastas já renomeadas. Serve como visão/upgrade; para demo precisa limpeza. Parcial
P0 técnico

Correções obrigatórias

  1. Criar ou remover setup-openclaw-pessoal-diretor.
  2. Corrigir privacidade da diretoria para default privado em todos os arquivos.
  3. Fazer init falhar ou avisar claramente se sobrar placeholder operacional.
  4. Atualizar docs pós-init para paths renomeados.
P0 pedagógico

Correções de narrativa

  1. Reposicionar pessoal como “para todos”, não só diretor.
  2. Marcar diretoria como bloco pulável para funcionário.
  3. Separar “faça agora” de “próximo nível”.
  4. Prometer instalação progressiva, não arquitetura final no primeiro dia.
